XSS (Cross-Site Scripting) Уязвимость

Поле описания услуги не санитизируется и позволяет выполнять произвольный JavaScript код.

Примеры атак:

  • Отображение произвольного контента: <script>alert('XSS!')</script>
  • Кража сессии: <script>fetch('/steal-session', {method: 'POST', body: document.cookie})</script>
  • Автоматические действия: <script>fetch('/approve-service-request/1/', {method: 'POST'})</script>

Рекомендации по исправлению:

  • Использовать Django template filter |escape
  • Применять библиотеки для санитизации HTML (bleach)
  • Настроить Content Security Policy (CSP)
  • Валидировать пользовательский ввод на сервере
